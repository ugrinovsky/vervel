# Vervel — Техническое задание

> Фитнес-платформа для тренеров и атлетов с AI-функциями, аналитикой нагрузок и системой коммуникации.

---

## 0. Описание продукта

**Vervel** — связка инструментов для учёта тренировок, наглядной обратной связи по нагрузке (2D-аватар, зоны тела), планирования и общения тренер↔атлет, с опциональным AI и монетизацией AI через внутренний баланс.

### Роли

| Роль | Назначение |
|------|------------|
| `athlete` | Свои тренировки, аналитика, команда, чаты с тренером |
| `trainer` | Атлеты, группы, календарь, шаблоны, данные по атлетам |
| `both` | Обе роли; в UI переключается активный кабинет (тренер / атлет) |

### Атлет

- Календарь активности, детали тренировок, черновик тренировки.
- Создание тренировки вручную; **AI**: распознавание по фото, генерация по тексту, парсинг заметок и применение к тренировке.
- Экран **Avatar** — нагрузка и восстановление по зонам (модель с затуханием).
- **Аналитика** — период, зоны, графики.
- **Прогрессия** и **силовой дневник** (`/strength-log`).
- **Периодизация** — ATL / CTL / TSB (PMC, Banister).
- **Серии и достижения** — streak, история, разблокировка и просмотр ачивок.
- **Команда** — группы и тренеры, инвайты (ссылка / QR), публичный профиль тренера.
- **Лидерборд** по группе.
- **Диалоги** — список чатов, сообщения, поток обновлений, отметка прочитанного.
- **Входящий видеозвонок** — опрос, приём/отклонение, LiveKit.
- **Профиль** — данные, фото, пароль, смена роли, замеры тела, тема, кошелёк AI и пополнение (ЮKassa), реферальная статистика.

### Тренер

- Обзор **«Сегодня»**, календарь и плановые тренировки (CRUD).
- **Команда** — атлеты и группы; карточка атлета (статы, аватар нагрузки, периодизация, тренировки и др.), **видеозвонок** 1:1 или группе.
- **Шаблоны** тренировок и **библиотека упражнений**.
- **Чаты** — личные и групповые.
- История/управление **видеозвонками** (API).

### Общее

- **Авторизация:** email/пароль (rate limit), OAuth VK и Яндекс, выбор роли для новых OAuth-пользователей; доступ по httpOnly cookie `auth_token` (на сервере подстановка в Bearer для guard).
- **AI:** баланс, транзакции, статус; тарификация чата по токенам.
- **Платежи:** пополнение кошелька, вебхук ЮKassa.
- **Web Push:** VAPID, подписка/отписка.
- **Обратная связь** с привязкой к пользователю.
- **PWA:** service worker (офлайн-страница, push).

### Публичная часть

- Лендинг, вход/регистрация, OAuth callback, приглашение по ссылке, документы (privacy / offer / seller).

## 6. Сервисы бизнес-логики

### WorkoutCalculator
Расчёт нагрузки по зонам тела при создании тренировки.

- **bodybuilding**: `weight × reps`, нормализуется к `[0..1]`
- **crossfit**: `rounds / MAX_ROUNDS_FOR_FULL_LOAD`
- **cardio**: `duration / MAX_TIME (3600с)`
- **calculateRecoveryState** — декаментное затухание нагрузки (λ=0.3) для модели восстановления
- **calculatePeriodStats** — агрегация за период: объём, интенсивность, breakdown по типам

### AiBalanceService
Кошелёк для AI-функций.

- `charge(userId, amount, description)` — списание с транзакцией БД
- `calculateChatCost(inputTokens, outputTokens)` — стоимость = `tokens/1000 × rate × markup`, мин. `CHAT_MIN_CHARGE`
- `InsufficientBalanceError` — бросается при нехватке средств

### YandexAiService
- `recognizeFromImage` — OCR (Yandex Vision) → LLM-парсинг → структурированная тренировка
- `generateFromText` — описание → тренировка через YandexGPT
- `chat` — история сообщений → ответ + `{ inputTokens, outputTokens }`

### StreakService
- Логика streak: тот же день → без изменений; +1 день → `currentStreak++`; пропуск → `currentStreak=1`
- Логирует каждое событие в `streak_history`
- После обновления вызывает `AchievementService.checkAndUnlockAchievements`

### AchievementService
Разблокировка достижений по триггерам:
- `streak_days` — серия N дней
- `total_workouts` — всего N тренировок
- `ai_chat_messages` — N сообщений AI-чата
- `groups_joined` — состоит в N группах
- `trainer_messages` — N сообщений тренерам

### ExerciseCatalog
Статический каталог. Fuzzy-матчинг (fuzzball `token_set_ratio ≥ 72`) для AI → каталог.

## 7. Модель периодизации (PMC — Banister)

```
TL  = (intensity×0.7 + min(volume/5000, 1)×0.3) × 100   — нагрузка дня
ATL = ATL_prev × (6/7) + TL × (1/7)    — 7-дн. EWMA  (усталость)
CTL = CTL_prev × (41/42) + TL × (1/42) — 42-дн. EWMA (форма)
TSB = CTL - ATL                          — свежесть
```

| TSB | Фаза |
|-----|------|
| > +15 | Deload / пик готовности |
| 0…+15 | Оптимальная форма |
| −20…0 | Накопление / рост формы |
| < −20 | Перетренированность |

## 9. Оплата (ЮКасса)

1. Пользователь выбирает сумму (100/250/500/1000₽)
2. `POST /payments/topup` → создаёт платёж ЮКасса → возвращает `confirmationUrl`
3. Редирект на страницу оплаты ЮКасса
4. После оплаты → `POST /payments/webhook` (IP-whitelist ЮКасса)
5. Backend: `balance += amount`, запись в `balance_transactions type='topup'`
6. Frontend: `/profile?topup=success` → toast + обновление баланса

## 10. AI-флоу создания тренировки

```
Фото/текст → YandexAiService → AiWorkoutResult {
  workoutType, exercises[{ name(EN), displayName(RU), sets, reps, weight }]
}
↓
ExerciseCatalog.fuzzyMatch(name) → exerciseId из каталога
↓
WorkoutInlineForm (редактирование перед сохранением)
↓
POST /workouts → WorkoutCalculator → сохранение + streak
```
