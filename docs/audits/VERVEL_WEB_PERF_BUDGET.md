# Vervel — Web performance budget (Vite/React)

Дата: 2026-05-06  
Цель: зафиксировать минимальные “перф-границы”, чтобы initial bundle не раздувался незаметно и mobile-first вход не деградировал.

---

## 1) Бюджет (первый проход)

Эти значения — стартовые (реальные цифры лучше уточнить после 1–2 измерений `vite build`).

- **Initial JS (gzip)**: стремиться к \(< 350–450KB\) на первый экран (без “тяжёлых” тренерских/аналитических зон).
- **Крупные зоны** (trainer calendar/templates/library, analytics, dialogs, workout form): должны грузиться **лениво** (route-level split).
- **LCP** (мобильный mid-tier): цель \(< 2.5–3.0s\) на /home после логина.

---

## 2) Как измерять (ручной минимум)

### Bundle / chunks
- собрать web:

```bash
cd apps/web
npm run build
```

- проверить размеры чанков в `dist/assets/*` (и сравнить с прошлой сборкой).

### RUM/Perf
- хотя бы 1 прогон Lighthouse для `/` и `/home` на mobile profile (перед релизом).

---

## 3) Правила “чтобы не раздувать”

- Новые экраны в `routes.tsx` и публичные страницы в `App.tsx` добавлять **через lazy + Suspense** (как baseline).
- Тяжёлые компоненты (графики, редакторы, большие таблицы) держать в **отдельных чанках** и грузить по месту.
