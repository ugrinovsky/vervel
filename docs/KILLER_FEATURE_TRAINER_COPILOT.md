# Trainer Copilot / Ассистент тренера

Дата: 2026-05-08  
Статус: в разработке

---

## Суть в одной фразе

Тренер видит кому нужна помощь, нажимает одну кнопку — ИИ составляет программу на неделю с упражнениями и подходами, атлет получает её и сообщение в чат.

---

## Концепция

Тренер ведёт 10–30 атлетов. Каждую неделю нужно:
1. Посмотреть кто перегружен / недогружен / пропустил тренировки
2. Составить план на неделю для каждого (упражнения, подходы, веса)
3. Написать в чат объяснение

Это 20–40 минут рутины. Ассистент делает это за тренера за 30 секунд на атлета.

**Важно:** Ассистент — это не "фича ради фичи". Тренер получает готовую программу с реальными упражнениями и весами, которую он может отредактировать и назначить. Один вызов AI — один результат — одна оплата.

---

## Типы тренировок

### Самостоятельные тренировки (основной сценарий)
- Тренер составляет программу → назначает на день
- Атлет делает сам когда удобно, вносит результаты и комментарий
- Тренер видит выполнено / не выполнено, ставит реакцию
- Время тренировки не фиксируется — атлет сам решает когда

### Персональные тренировки (флаг)
- Флаг `isPersonal` на `ScheduledWorkout` — тренировка с тренером физически
- Атлет видит бейдж "с тренером"
- Запись/booking в первом этапе не строим, флаг выставляется вручную

## Поэтапный план реализации

---

### ✅ Этап 1: Ядро ассистента (реализовано)

**Backend:**
- `CopilotSharedRules.ts` — константы: пороги TSB, зон, volume adjustment
- `CopilotInsightsService.ts` — периодизация + перегруженные зоны + история
- `CopilotPlanService.ts` — один AI-вызов генерирует полную программу с упражнениями + сообщение (специализация определяется по шаблонам и истории назначений тренера)
- `CopilotPriorityService.ts` — ранжирование атлетов по срочности
- `POST /trainer/copilot/draft` — генерация черновика (стоимость: `AI_COST_COPILOT`)
- `POST /trainer/copilot/commit` — назначение тренировок + отправка сообщения в чат
- `GET /trainer/copilot/priority-list` — список атлетов: нуждаются во внимании / всё ок
- Определение специализации тренера: сначала шаблоны (≥3), потом история `ScheduledWorkout`

**Frontend:**
- `CopilotAthleteList` — список атлетов с бейджами срочности, фазы, усталости, "черновик готов"
- `CopilotPanel` — панель для атлета: статус, план по дням, редактор упражнений (BottomSheet), сообщение, кнопки "Назначить и отправить" / "Обновить" / "Удалить черновик"
- Черновик сохраняется в `localStorage` — не сбрасывается при навигации и перезагрузке
- Панель не размонтируется при сворачивании строки атлета (always-mounted)
- `PlanItem` — строка дня: дата, тип, акцент, кол-во упражнений, спойлер с упражнениями, кнопка редактирования
- Редактор упражнений по дню — полноценный `WorkoutExercisesEditor` в `BottomSheet`
- `TemplateListSheet` — управление шаблонами на той же странице без редиректа
- Переименовано: Copilot → Ассистент везде в UI

### 🔲 Этап 2: Обратная связь атлета (следующий)

Замыкает петлю: тренер видит не только "назначил", но и "выполнено / с каким результатом".

**Backend — миграция:**
```sql
ALTER TABLE scheduled_workouts
  ADD COLUMN completed_at TIMESTAMP NULL,
  ADD COLUMN athlete_comment TEXT NULL,
  ADD COLUMN trainer_reaction VARCHAR(10) NULL, -- эмодзи или null
  ADD COLUMN is_personal BOOLEAN NOT NULL DEFAULT FALSE;
```

**Backend — эндпоинты:**
- `PATCH /scheduled-workouts/:id/complete` — атлет отмечает выполненным + комментарий
- `PATCH /scheduled-workouts/:id/react` — тренер ставит реакцию (👍 и т.п.)

**Frontend (атлет):**
- На карточке назначенной тренировки (в `AthleteCopilotCard` / списке недели): кнопка "Отметить выполненным"
- Тап открывает мини-шит: поле для комментария (опционально) + кнопка подтверждения
- После — карточка показывает "✓ Выполнено" + текст комментария

**Frontend (тренер):**
- В `CopilotAthleteList`: у строки атлета показывать количество выполненных / невыполненных за неделю
- В `CopilotPanel` (или отдельной вкладке): список выполненных с комментариями, кнопка реакции рядом
- Пуш тренеру когда атлет выполнил назначенную тренировку

---

### 🔲 Этап 3: Качество плана

**Настройка количества тренировок:**
- Сейчас захардкожено `sessionsPerWeek: 3`
- Добавить в `CopilotPanel` простой степпер: 2 / 3 / 4 / 5 тренировок в неделю
- Сохранять последнее значение в `localStorage` на атлета

**Цель атлета:**
- Добавить выбор цели в панели: Сила / Выносливость / Смешанная
- Передавать в `copilotDraft` как `goal`

**Throttle:**
- Не более 3 `draft` за 10 минут на одного атлета
- Защита от случайного биллинга при многократных нажатиях "Обновить"

---

### 🔲 Этап 4: Автоматизация

**Авто-followup:**
- Если атлет не выполнил назначение за 48 часов — напоминание тренеру в приложении
- Cron-задача: проверяет `ScheduledWorkout` где `status=scheduled` и дата прошла > 48ч

**Пуш тренеру:**
- Когда атлет начинает назначенную тренировку ("Иван начал тренировку")
- Когда атлет выполнил и оставил комментарий

**Пуш атлету:**
- Напоминание утром в день назначенной тренировки

## Архитектура

```
AI-вызов (один на draft):
  YandexAiService.generateCopilotPlan()
    ← TSB, фаза, перегруженные зоны, специализация тренера, sessionsPerWeek, goal
    → plan[] (дни с упражнениями) + chatMessage

Специализация тренера (без AI):
  WorkoutTemplate (тип по большинству) → если < 3 шаблонов → ScheduledWorkout история

Стоимость:
  AI_COST_COPILOT (env, дефолт 8₽) — одно списание за draft
  Повторный draft = повторное списание (тренер видит стоимость)
```

```
Сервисный слой:
  CopilotSharedRules       — константы: пороги TSB/зон, volume adj
  CopilotInsightsService   — периодизация + зоны + история
  CopilotPlanService       — AI-генерация программы + определение специализации
  CopilotPriorityService   — ранжирование атлетов по срочности
  YandexAiService          — generateCopilotPlan() метод
```

## Flywheel: как ассистент становится умнее со временем

```
Пн утро: тренер открывает ассистента
  → видит "Иван: Перегрузка, 5 дней без плана"
  → нажимает "Составить программу"
  → ИИ генерирует: пн верх тела, ср ноги, пт кардио (учёл перегруженные плечи)
  → тренер редактирует пт (меняет кардио на функциональную)
  → нажимает "Назначить и отправить"

Пн вечер: Иван выполнил тренировку
  → нажимает "Выполнено" + "Всё отлично, вес взял нормально"
  → тренер получает пуш, ставит 👍

Следующий пн: ассистент учитывает
  → zonesLoadAbs обновились → точнее определяет перегруженные зоны
  → daysSinceLastWorkout = 0 → не ставит "без тренировки"
  → видит выполнение 3/3 на прошлой неделе → план с полным объёмом
```

## Риски

| Риск | Митигация |
|---|---|
| ИИ генерирует нерелевантный тип тренировки | Специализация тренера передаётся как жёсткое ограничение в промпт |
| Тренер без шаблонов и истории → неизвестная специализация | Fallback: bodybuilding (самый универсальный); в будущем — поле в профиле тренера |
| Случайный повторный draft = повторная оплата | Throttle в этапе 3; сейчас — видимая стоимость перед действием |
| Атлет уже имеет план на неделю | `existingOffsets` фильтруются, AI не дублирует занятые дни |
| Черновик сбросился | `localStorage` — не сбрасывается без явного "Удалить черновик" |
